# 网页设计系统 (Website Design System)

## 概述

本技能为前端开发者和设计师提供**经过精心设计的设计系统模板**，每套模板都包含完整的设计哲学、设计 Token、组件样式和响应式策略。这些不是普通的 UI 模板，而是能够让你的网站脱颖而出、避免"AI 生成感"的独特风格指南。

## 适用场景

- 🎨 需要为新项目选择设计风格
- 🔧 将设计系统集成到现有代码库
- 📐 需要完整的设计规范（色彩、字体、间距、动画）
- 🚀 想要快速构建高质量、非通用化的 UI

## 可用风格

| 风格 | 描述 | 适合场景 |
|------|------|----------|
| **Monochrome** | 极简黑白风格，纯粹的黑与白、大字号、线条分割 | 高端品牌、时尚杂志、艺术画廊、奢侈品网站 |
| **Bauhaus** | 包豪斯几何风格，原色（红蓝黄）、硬阴影、几何图形 | 创意机构、设计工作室、艺术展览、建筑类网站 |

## 使用方法

### 查阅设计规范

每套设计系统都包含完整的实现指南，请根据需要阅读对应文档：

```python
# Monochrome 极简黑白风格
skills_read(path="skills/website_design/docs/monochrome.md")

# Bauhaus 包豪斯几何风格
skills_read(path="skills/website_design/docs/bauhaus.md")
```

### 文档结构

每份设计文档包含以下内容：

1. **设计哲学** - 风格的核心理念和视觉关键词
2. **设计 Token 系统** - 颜色、字体、间距、圆角、阴影等设计变量
3. **组件样式** - 按钮、卡片、输入框等常用组件的具体样式
4. **布局策略** - 容器宽度、网格系统、响应式断点
5. **动画与交互** - 过渡效果、悬停状态、焦点样式
6. **无障碍设计** - 对比度、焦点状态、触摸目标等
7. **非通用化要点** - 避免"模板感"的关键设计决策

## 风格速览

### Monochrome 极简黑白

```
颜色：纯黑 #000000 + 纯白 #FFFFFF（无灰度主色）
字体：Playfair Display（衬线体）+ Source Serif 4
圆角：0px（全部锐角）
阴影：无
特点：大字号标题、线条分割、色彩反转强调、编辑式排版
```

### Bauhaus 包豪斯

```
颜色：红 #D02020 + 蓝 #1040C0 + 黄 #F0C020 + 黑 #121212
字体：Outfit（几何无衬线体）
圆角：0px 或 100%（二元极端）
阴影：硬阴影 4px/8px 偏移
特点：几何图形装饰、色块背景、45° 旋转元素、构成主义
```

## 注意事项

1. **阅读完整文档**：每份设计文档都包含重要的"非通用化要点"，跳过可能导致设计失去独特性
2. **保持一致性**：选定风格后，应在整个项目中保持一致
3. **理解用户目标**：在实施前，先与用户确认是要重设计组件、重构现有样式还是全新构建
4. **响应式优先**：所有设计规范都考虑了移动端适配
